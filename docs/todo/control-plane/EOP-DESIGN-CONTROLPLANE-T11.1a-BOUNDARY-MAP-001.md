## 1. Method

Classify every top-level module in `rust/src/` (the `ob-poc` binary crate, ~278K lines, 36 top-level entries) into: **agent-tier** (interpretation/conversational — target: zero capability-crate dependency edges except via `ob-poc-control-plane`), **capability-tier** (owns DB/execution — stays where it is, becomes a keyed-door target in T11.2), or **CP-adjacent** (glue/routing/infrastructure that touches both sides but isn't itself interpretation or execution logic — e.g. HTTP routing, session-state plumbing).

Evidence: `grep -rl "sqlx::\|dsl_runtime::\|sem_os_postgres::"` per module directory (a rough capability-tier signal, not a proof of purpose) plus a purpose read of each module's own doc comment / `mod.rs`. Full census run via Explore agent, spot-verified directly for the two most consequential/surprising findings (`sage/mod.rs`, `constellation/`).

**Where a module mixes conversational/interpretation code with direct capability access in the same directory ("MIXED"), the readership rule (ledger, ratified per E-3: *a write is operational if any capability/gate/audit-of-record path reads it; agent-local only if the agent tier is the sole consumer*) is the classification test — applied at whatever grain the evidence supports. Four modules are large enough (1,000+ lines, dozens of files) that a confident file-level split is not responsibly claimable from a directory-level grep; those are flagged for a dedicated finer-grained pass, not force-classified here.**
